## 4. 多语言发布策略

### 4.1 发布顺序

1. **中文（zh）**：MVP 首发，目标市场中国大陆、港澳台及海外华人。内容最完善。
2. **英文（en）**：第二优先级，上线后 1-2 周内完成。目标是 Product Hunt、Hacker News、Reddit 等英文社区。

### 4.2 翻译策略

- **页面级翻译**：每种语言独立生成静态 HTML，不依赖前端 JS 替换，确保搜索引擎收录完整页面内容。
- **SEO 元数据独立**：每语种的 `title`、`description`、`OG`、`Twitter Card`、`JSON-LD` 均独立撰写，不做机械翻译。
- **URL 策略**：使用路径区分 `/zh/`、`/en/`，根路径 `/` 根据浏览器语言自动跳转（优先中文，其次英文）。
- **关键词本地化**：
  - 中文：AI聚合、多窗口工具、效率导航、网站聚合面板
  - 英文：AI aggregation, multi-window dashboard, productivity tools, site aggregator

### 4.3 hreflang 检查清单

- [ ] 每页头部包含指向自身和另一语言的 `hreflang` 标签。
- [ ] 包含 `hreflang="x-default"` 指向默认语言（中文）。
- [ ] `sitemap.xml` 中使用 `<xhtml:link>` 声明多语言关系。
- [ ] 使用 Google Search Console 的"国际定位"报告检查是否有 hreflang 错误。

### 4.4 各语言版本推广入口

| 语言 | 核心推广入口 |
|------|-------------|
| 中文 | 知乎、微信公众号、小红书、V2EX、即刻、少数派 |
| 英文 | Product Hunt、Reddit、Hacker News、Twitter/X、Medium、Indie Hackers |

## 5. Cloudflare Pages 部署与 DNS

### 5.1 Cloudflare Pages 设置步骤

1. 将代码推送到 GitHub 仓库（如 `GITHUB_OWNER/ai-dialogue-aggregation`）。
2. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)，进入 **Pages**。
3. 点击 **Create a project** → **Connect to Git**。
4. 选择你的 GitHub 仓库，授权 Cloudflare 访问。
5. 构建设置（静态站点无需复杂配置）：
   - Build command:（留空，或填 `echo "static"`）
   - Build output directory: `/`
6. 点击 **Save and Deploy**，Cloudflare 会自动构建并分配一个 `*.pages.dev` 域名。

### 5.2 自动部署（GitHub Actions）

本项目已包含 `.github/workflows/deploy.yml`，使用官方 `cloudflare/pages-action@v1`。

**需要在 GitHub 仓库中配置以下 Secrets**（Settings > Secrets and variables > Actions）：

| Secret | 获取方式 |
|--------|---------|
| `CLOUDFLARE_API_TOKEN` | Cloudflare 控制台 → My Profile → API Tokens → Create Token → 使用 **Cloudflare Pages: Edit** 模板 |
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare 控制台首页右侧的 Account ID |
| `CLOUDFLARE_PROJECT_NAME` | 你在 Cloudflare Pages 创建的项目名称 |

**Token 权限要求**：
- Account: Cloudflare Pages: Edit
- Zone: 如需自动管理 DNS，可额外添加 Zone:Read + DNS:Edit

### 5.3 自定义域名绑定

1. 进入 Cloudflare Pages 项目 → **Custom domains**。
2. 输入你的域名（如 `aichatagg.com` 或 `www.aichatagg.com`）。
3. Cloudflare 会自动检测 DNS 配置。如果域名托管在 Cloudflare，通常一步完成；如果托管在第三方（如 Dynadot），需要手动添加 CNAME 记录。

**DNS 配置（域名托管在 Cloudflare 时）**：

| 类型 | 名称 | 目标 | 代理状态 | 说明 |
|------|------|------|---------|------|
| CNAME | `www` | `your-project.pages.dev` | 已代理 | www 子域名 |
| CNAME | `@` (根域名) | `your-project.pages.dev` | 已代理 | Cloudflare 支持根域名 CNAME Flattening |

**DNS 配置（域名托管在 Dynadot 等其他服务商时）**：
- 添加 CNAME 记录：`www` → `your-project.pages.dev`
- 根域名（`@`）无法直接 CNAME 时，使用 Cloudflare 的 DNS 服务（推荐将域名 NS 转移到 Cloudflare），或使用转发规则将根域名 301 跳转到 `www`。

### 5.4 自动部署脚本示例（本地备用）

如果使用 Wrangler CLI 手动部署：

```bash
# 安装 wrangler
npm install -g wrangler

# 登录 Cloudflare
wrangler login

# 部署当前目录
wrangler pages deploy . --project-name=ai-dialogue-aggregation
```

**安全要求**：
- API Token 从 GitHub Secrets 读取，绝不写入仓库代码。
- 本地部署时，`wrangler login` 使用浏览器 OAuth，不保存密码。

### 5.5 回滚方案

| 场景 | 回滚操作 |
|------|---------|
| 代码回滚 | `git revert` 或强制回退 `main` 分支到上一个稳定 commit，GitHub Actions 会自动重新部署旧版本到 Cloudflare Pages。 |
| Cloudflare Pages 回滚 | 进入 Cloudflare Pages 项目 → Deployments → 找到上一个稳定版本 → 点击 **Rollback**。 |
| DNS 回滚 | 在 Cloudflare DNS 面板恢复旧记录；如果域名在第三方，登录对应平台修改。 |
| 域名解析故障 | 快速切回 `*.pages.dev` 原始域名，确认是代码问题还是 DNS 问题。 |

### 5.6 验证方式

- [ ] 访问 `https://your-project.pages.dev` 和自定义域名均能正常显示，且证书有效。
- [ ] 使用 `dig www.your-domain.com +short` 确认解析到 Cloudflare。
- [ ] 使用 [SSL Labs](https://www.ssllabs.com/ssltest/) 检查 HTTPS 配置评级（Cloudflare 默认 A+）。
- [ ] 使用 [Google Rich Results Test](https://search.google.com/test/rich-results) 验证结构化数据。
- [ ] 在 Cloudflare Pages 控制台确认最新 Deployment 状态为 Success。

## 6. 流量监测方案

| 工具 | 用途 | 部署方式 |

| **Google Analytics 4 (GA4)** | 页面浏览、用户行为、流量来源、留存分析 | 在 `<head>` 中嵌入 gtag.js 脚本（需替换为实际 Measurement ID） |

## 8. 后续迭代建议与风险提醒

### 8.1 迭代建议（按优先级）

1. **隐私政策页**：已上线 `/zh/privacy.html` 和 `/en/privacy.html`，这是 AdSense 和合规的硬性要求。✅
2. **浏览器扩展**：开发 Chrome/Edge 扩展，实现真正的"选中网页文字 -> 右键发送到多个AI"，绕过 iframe 限制。
3. **PWA 支持**：添加 `manifest.json` 和 Service Worker，让用户可以"安装"为桌面/手机应用。
4. **深色/浅色模式切换按钮**：目前依赖系统偏好，增加手动切换可提升用户体验。
5. **窗口大小记忆**：将用户拖拽调整的窗口尺寸保存到 localStorage，刷新后保持自定义布局。
6. **云端配置同步**：后端使用无服务器方案（如 Cloudflare Workers + KV）保存用户配置，实现跨设备同步。
7. **AI 回答对比评分**：让用户对同一问题的不同 AI 回答进行打分，生成社区排行榜。

### 8.2 风险提醒

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| **iframe CSP 限制** | 高 | 多数主流 AI 站（ChatGPT、DeepSeek 等）禁止 iframe 嵌入。已在前端提示用户，长期应转向浏览器扩展方案。 |
| **域名/品牌被抢注** | 中 | 尽快注册首选域名和核心变体，同步申请商标（如预算允许）。 |
| **搜索引擎不收录动态少页面** | 中 | 已增加静态内容区块和隐私政策页；持续增加内容页保持更新频率。 |
| **AdSense 审核不通过** | 中 | 严格满足"内容质量 + 隐私政策 + 自然流量"三大条件后再申请，避免频繁被拒导致账号标记。 |
| **第三方网站 URL 变更** | 低 | 预设列表中的 URL 失效时，用户可通过"切换网站"功能自行更换；定期检查并更新默认预设。 |
| **Cloudflare Pages 构建失败** | 低 | 纯静态站点构建失败概率极低；如失败可在 Cloudflare 控制台查看构建日志并修复。 |
